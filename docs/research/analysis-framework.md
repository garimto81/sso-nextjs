# 분석 프레임워크

이 문서는 GitHub 리포지토리를 체계적으로 분석하기 위한 방법론과 평가 기준을 정의합니다.

---

## 분석 항목

### 1. 기본 정보
- GitHub Stars 수
- Last Commit 날짜
- Contributors 수
- 라이선스
- 문서화 수준

### 2. 기술 스택
- Next.js 버전 (13, 14, 15)
- 인증 솔루션 (NextAuth v5, Supabase Auth, 기타)
- 데이터베이스 (Prisma, Supabase DB, PostgreSQL 등)
- UI 라이브러리 (Tailwind, Shadcn UI, 기타)
- 유효성 검사 (Zod, Yup, 기타)
- 배포 플랫폼 (Vercel, 기타)

### 3. 아키텍처 분석
- 폴더 구조 패턴
- App Router vs Pages Router
- Server Components vs Client Components 사용 패턴
- API Routes vs Server Actions
- 미들웨어 구현 방식
- 상태 관리 패턴

### 4. 인증 흐름
- 인증 제공자 (Credentials, OAuth, Magic Link 등)
- 세션 관리 방식 (JWT, Database Session)
- 토큰 저장 방식 (httpOnly cookie, localStorage 등)
- 세션 갱신 로직
- 로그아웃 처리

### 5. 보안 구현
- CSRF 보호
- XSS 방지
- Rate Limiting
- 계정 잠금 (Account Lockout)
- 비밀번호 정책
- RLS (Row Level Security) - Supabase 전용
- 환경 변수 관리

### 6. 데이터베이스 스키마
- 테이블 설계
- 관계 정의
- 마이그레이션 전략
- 시드 데이터

### 7. 테스트 전략
- 유닛 테스트 (Jest, Vitest 등)
- 통합 테스트
- E2E 테스트 (Playwright, Cypress 등)
- 테스트 커버리지
- CI/CD 통합

### 8. 코드 품질
- TypeScript 사용도
- 타입 안정성
- ESLint 규칙
- Prettier 설정
- 코드 구조화 (재사용성, 모듈화)

### 9. 성능 최적화
- 이미지 최적화
- 코드 스플리팅
- Lazy Loading
- 캐싱 전략
- 번들 사이즈

### 10. 사용자 경험 (UX)
- 로딩 상태 표시
- 에러 처리 UI
- 폼 유효성 검사 피드백
- 접근성 (a11y)
- 반응형 디자인

## 평가 기준

각 항목을 1-5점 척도로 평가:

### 적용 용이성 (Ease of Adoption)
- **5점**: 즉시 복사 가능, 최소한의 수정
- **4점**: 1-2시간 내 적용 가능
- **3점**: 하루 작업으로 적용 가능
- **2점**: 2-3일 소요, 중간 난이도
- **1점**: 1주일 이상 소요, 큰 리팩토링 필요

### PRD 적합성 (PRD Alignment)
- **5점**: PRD 요구사항과 완벽히 일치
- **4점**: 대부분의 요구사항 충족
- **3점**: 일부 요구사항 충족
- **2점**: 간접적 관련성만 있음
- **1점**: PRD와 무관

### 유지보수성 (Maintainability)
- **5점**: 매우 명확한 코드, 훌륭한 문서화
- **4점**: 이해하기 쉬운 코드, 적절한 문서
- **3점**: 표준적인 코드, 기본 문서
- **2점**: 복잡한 코드, 문서 부족
- **1점**: 레거시 패턴, 문서 없음

### 커뮤니티 활성도 (Community Activity)
- **5점**: 주간 업데이트, 활발한 이슈 해결
- **4점**: 월간 업데이트, 이슈 관리 양호
- **3점**: 분기별 업데이트
- **2점**: 연간 1-2회 업데이트
- **1점**: 1년 이상 업데이트 없음

### 확장성 (Scalability)
- **5점**: 엔터프라이즈급 확장성
- **4점**: 중대형 앱에 적합
- **3점**: 중소형 앱에 적합
- **2점**: 프로토타입/MVP 수준
- **1점**: 데모 용도만

## 우리 프로젝트와 비교 템플릿

각 분석 문서에는 다음 비교표 포함:

| 항목 | 우리 프로젝트 | 분석 대상 | 차이점 | 우선순위 |
|------|--------------|-----------|--------|----------|
| NextAuth v5 | ✅ Credentials | ✅ + OAuth | OAuth 미구현 | 🟡 Medium |
| 미들웨어 | ❌ | ✅ | 라우트 보호 없음 | 🔴 High |
| Supabase | ❌ | ✅ | 플레이스홀더 인증 | 🔴 High |
| Rate Limiting | ❌ | ❌ | 양쪽 다 없음 | 🟡 Medium |
| E2E 테스트 | ✅ Playwright | ❌ | 우리가 우수 | - |

**우선순위 범례**:
- 🔴 High - Phase 2 이전에 구현 필요
- 🟡 Medium - Phase 3-4에서 구현
- 🟢 Low - Phase 5+ 검토
- ⚪ Optional - Nice to have

## 개선 제안 추출 방법

### 1. 즉시 적용 가능 (Quick Wins)
- 설정 파일 (next.config.js, tsconfig.json 등)
- 유틸리티 함수
- 타입 정의
- ESLint/Prettier 규칙

### 2. 단기 적용 (Phase 2-3)
- 미들웨어 패턴
- 에러 처리 방식
- Supabase 통합 패턴
- 폴더 구조 개선

### 3. 중기 적용 (Phase 4-5)
- UI 컴포넌트 라이브러리
- 테스트 전략
- 성능 최적화 기법
- OAuth 제공자 추가

### 4. 장기 검토 (Phase 6+)
- 아키텍처 변경
- 데이터베이스 마이그레이션
- 모니터링/로깅 시스템
- 배포 파이프라인
